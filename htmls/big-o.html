Big-O 표기법이란?

Big-O 표기법은 알고리즘의 성능(복잡도)를 표현하는 수학적 표기입니다.
입력 크기 n이 커질 때, 알고리즘이 얼마나 빠르게(또는 느리게) 성장하는지를 나타냅니다.

🧠 시간 복잡도(Time Complexity)

입력 크기 n에 따라 연산 횟수가 얼마나 증가하는지를 나타냅니다.
일반적인 시간 복잡도 유형은 다음과 같습니다:

복잡도 명칭 설명 예시
O(1) 상수 시간 입력 크기와 관계없이 일정한 시간에 수행됨 배열의 첫 번째 원소 접근
O(log n) 로그 시간 입력이 커질수록 비교 횟수가 로그 단위로 증가 이진 탐색
O(n) 선형 시간 입력 크기에 비례하여 실행 시간 증가 단순 탐색
O(n log n) 로그 선형 시간 효율적인 정렬 알고리즘의 전형적인 복잡도 병합 정렬, 퀵 정렬, 힙 정렬
O(n²) 제곱 시간 입력 크기가 두 배가 되면 실행 시간이 네 배로 증가 버블, 선택, 삽입 정렬
O(2ⁿ) 지수 시간 입력 크기가 조금만 커져도 실행 시간이 폭발적으로 증가 부분집합 탐색, 재귀 조합 문제

📉 공간 복잡도(Space Complexity)

알고리즘이 실행 중 사용하는 메모리 공간의 양을 나타냅니다. 예를 들어, 병합 정렬은 추가 배열이 필요하여 공간 복잡도가 O(n)이고, 힙 정렬은 배열 내부에서 수행되므로 O(1)에 가깝습니다.

💡 요약: Big-O는 코드의 효율성을 수학적으로 표현하는 도구로, 알고리즘의 성능 차이를 정량적으로 이해할 수 있게 해줍니다.
